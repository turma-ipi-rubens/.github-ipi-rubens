## ⚖️ Regras de Uso (Scrum Framework)

Para garantir a organização e o aprendizado, todos os grupos devem seguir estas diretrizes:

1. **Gestão Visual:** O quadro do **GitHub Projects** deve ser a "única fonte da verdade". Se uma tarefa não está lá, ela não existe.
2. **Histórias de Usuário:** As tarefas devem ser escritas conforme exemplo: [US01]-Nome da História.
3. **Tasks:** As tarefas deverão ser escritas conforme o exemplo: [US01][BD][T01] - Nome da task.
4. **Sprints:** As iterações são semanais. O planejamento deve ocorrer na segunda-feira e a entrega (Review) conforme calendário acadêmico.
5. **Código e Git:** - Use ramos (branches) para novas funcionalidades.
   - Nomeie commits de forma clara.
   - Utilize *Pull Requests* para revisão de código entre os pares.

## 🎓 Avaliação e Monitoramento

O professor (**Sponsor**) acompanhará o progresso através de:
- **Insights do Projeto:** Gráficos de Burn-up e velocidade da equipe.
- **Histórico de Commits:** Frequência e qualidade das contribuições individuais.
- **Critérios de Aceite:** Validação funcional das entregas ao final de cada Sprint.
